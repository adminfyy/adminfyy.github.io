# Claude Code 在大型前端仓库的调试与重构方法

大型仓库里，Claude Code 的难点不是“写功能”，而是“定位问题与控制影响面”。

## 1. 调试前先做上下文收敛

建议先收敛三件事：

1. 失败症状：报错日志、复现步骤、影响范围
2. 最近变更：最近提交、依赖升级、配置调整
3. 关键路径：触发问题的路由与模块链路

没有收敛，调试会变成盲目搜索。

## 2. 系统化调试步骤

1. 先复现（最小路径）
2. 再观测（日志、埋点、断点）
3. 再定位（调用链、状态链、渲染链）
4. 再修复（最小改动）
5. 再回归（关联场景）

## 3. 重构策略：分层替换而非整块重写

推荐顺序：

1. 提取纯函数与类型
2. 收敛副作用（请求、订阅、定时器）
3. 拆分巨型组件
4. 补测试后再继续抽象

## 4. 风险控制清单

- 是否改变对外接口
- 是否影响路由/权限/缓存
- 是否增加首屏或主包体积
- 是否引入跨模块循环依赖

## 5. 典型 Prompt

```text
目标：修复某页面内存上涨问题并做最小重构
范围：src/views/x, src/stores/y
约束：不改 API 协议，不引入新依赖
验收：测试通过 + build 通过 + 复现路径稳定
```

## 6. 结论

在大仓库里，Claude Code 的最佳角色是“可控执行器”：  
先定位，再小步改动，再证据化验证。
