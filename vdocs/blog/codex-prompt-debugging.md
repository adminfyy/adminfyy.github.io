# Codex Prompt 调试手册：从“说不清”到“可执行”

很多人觉得 Codex 不稳定，实际问题常常不是模型能力，而是 Prompt 不可执行。

## 1. 典型坏味道

- 目标模糊：比如“帮我优化一下项目”
- 没有边界：没有限定文件范围、技术栈、验收标准
- 缺失验证：没有要求测试、构建或回归步骤

## 2. 可执行 Prompt 模板

```text
目标：
技术栈：
改动范围（文件/模块）：
约束：
验收标准：
输出格式：
```

## 3. 调试方法

1. 先缩小任务范围，只给一个文件或一个路由。
2. 要求输出“变更点列表”，不要直接大段解释。
3. 强制要求验证命令和结果摘要。
4. 如果第一次跑偏，追加“反例约束”。

## 4. 实战示例

```text
目标：新增 /blog/:slug 详情页并接入首页
技术栈：Vue2 + Vite + Vue Router
改动范围：src/views/HomeView.vue, src/views/BlogDetailView.vue, src/router/index.ts
约束：不改接口层，不引入第三方包
验收标准：npm run build-only 通过，首页可跳详情页
输出格式：先给变更计划，再给代码改动摘要
```

## 5. 结论

Prompt 质量 = 任务清晰度。  
当输入是“结构化任务单”，Codex 才会稳定输出“工程结果”。
