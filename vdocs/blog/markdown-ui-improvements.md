## 🎯 设计原则

### 1. **一致性**
- 统一使用主题色 `#8de8ff` 和 `#61daff`
- 所有卡片使用相同的圆角和边框样式
- 保持统一的间距和字体大小

### 2. **层次感**
- 通过阴影和边框创建视觉层次
- 使用渐变增加深度感
- 标题层级清晰可辨

### 3. **交互反馈**
- 所有可交互元素都有悬停效果
- 使用平滑的过渡动画（0.2s - 0.3s）
- 提供明确的视觉反馈

### 4. **可读性**
- 合适的行高（1.6 - 1.8）
- 足够的对比度
- 合理的字体大小和间距

### 5. **响应式**
- 移动端友好的布局
- 自适应的字体大小（clamp）
- 灵活的网格系统

---

## 📱 响应式断点

- **桌面端**: ≥ 960px - 多列布局
- **平板端**: 640px - 960px - 调整列数
- **移动端**: < 640px - 单列布局

## 🎨 配色方案

### 主色调
- 主色: `#8de8ff` (浅蓝)
- 辅色: `#61daff` (天蓝)
- 强调色: `rgba(141, 232, 255, 0.15)` (半透明)

### 背景色
- 深色背景: `rgba(3, 13, 30, 0.5)`
- 玻璃态: `var(--surface-glass)`
- 代码块: `rgba(8, 22, 42, 0.8)`

### 文字色
- 标题: `var(--color-heading)` (#e8f7ff)
- 正文: `var(--color-text)` (#d7e8f8)
- 次要: `#93acc0`

## 📝 使用建议

### 对于博客作者：
1. 使用标准 Markdown 语法编写内容
2. 适当使用标题层级（H1-H4）
3. 善用代码块展示技术内容
4. 使用任务列表追踪进度
5. 添加表格对比数据

### 对于开发者：
1. 可以通过修改 CSS 变量快速调整主题
2. 所有样式都使用了 scoped，不会影响全局
3. Markdown 渲染器可以轻松扩展新功能
4. 动画时长可以在 CSS 中统一调整
