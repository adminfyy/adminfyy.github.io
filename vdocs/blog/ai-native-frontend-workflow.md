# AI Native 前端工作流：把 Codex/Agent 变成稳定产能

AI 不该只用来“生成代码片段”，而应进入完整交付链路。

## 1. 推荐工作流

1. 需求结构化：目标、范围、约束、验收
2. Agent 执行：先上下文扫描，再分步改动
3. 自动验证：lint/test/build 全跑
4. 交付汇总：变更文件、风险、回滚方案

## 2. Prompt 不是“提问”，是“任务协议”

一个可执行 Prompt 至少包含：

- 目标（做成什么）
- 边界（能改什么）
- 约束（不能做什么）
- 验收（怎么判断完成）
- 输出格式（你希望它如何汇报）

## 3. 架构层面怎么接入 AI

- 团队级模板：统一 Prompt 模板和输出模板
- CI 门禁：AI 生成代码也必须过同样的检查
- 风险分级：高风险变更强制人工二次评审

## 4. 哲学：人机协作，不是人机替代

- 人负责判断与取舍
- Agent 负责执行与提速
- 质量标准不因 AI 降级

## 5. 实战建议

1. 从低风险任务试点：样式、文档、测试补齐
2. 再扩展到中风险任务：页面模块、路由、状态
3. 最后再进入高风险任务：架构重构、核心链路

## 6. 结论

AI Native 团队的核心能力，不是“会不会用工具”，而是“有没有把工具纳入工程系统”。
