# Codex Agent 工作流：需求到交付的最短路径

把 Codex 当“助手”容易失控，把它当“执行代理”会更稳定。

## 一条推荐工作流

1. 定义目标：一句话说明最终可验收结果
2. 锁定范围：明确可改文件与不可改边界
3. 分步执行：先改数据层，再改页面，再跑验证
4. 汇总证据：输出命令结果与关键 diff

## 为什么有效

- 任务切片后，错误更容易回滚
- 每一步都有可观测结果（测试/构建）
- 最终输出更贴近 PR 描述格式

## 模板

```text
Step 1: Gather context
Step 2: Implement changes
Step 3: Verify (test/build)
Step 4: Summarize with file list
```

## 实战提醒

- 不要一次塞进“重构+新功能+样式大改”
- 先通路再优化，先可用再优雅
- 每一步都保留可回滚节点
