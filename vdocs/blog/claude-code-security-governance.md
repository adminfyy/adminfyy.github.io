# Claude Code 安全边界与风险治理：团队落地必读

AI 编码提速很快，但没有治理就会放大风险。

## 1. 三类核心风险

1. 代码风险：错误逻辑、回归缺陷、性能退化
2. 数据风险：敏感信息泄露、日志暴露
3. 流程风险：越权改动、不可追溯交付

## 2. 权限模型建议

- 读权限默认开放
- 写权限按目录和任务分级
- 高风险目录（鉴权、支付、安全）强制人工批准

## 3. 任务分级治理

- L1 低风险：样式、文档、测试补齐
- L2 中风险：页面逻辑、状态流改造
- L3 高风险：核心链路、鉴权、交易流程

L3 必须双人评审 + 全量回归。

## 4. 安全实践清单

1. Prompt 不直接写真实密钥与隐私数据
2. 输出中自动扫描敏感串（token/key/secret）
3. CI 强制 lint/test/build 门禁
4. 关键改动记录 ADR 与回滚方案

## 5. 可追溯交付要求

每次 AI 协作都要留下：

- 输入任务定义
- 变更文件列表
- 验证命令与结果
- 风险与回滚说明

这决定了问题发生时是否能快速止损。

## 6. 结论

Claude Code 的治理目标不是“限制效率”，而是“把效率变成可控收益”。
